# プログラミング学習システム 詳細仕様書 (Phase 1: MVP)

## 1. プロジェクト概要
本システムは、ブラウザ完結型の学習環境、AIによる動的な問題生成およびヒント提示、そしてゲームの「スキルツリー」のようなタグ熟練度システムを用いた、自己主導型のプログラミング学習プラットフォームである。

## 2. システム・アーキテクチャ要件
* **フロントエンド**: SPA (React / Vue.js等) - 画面左側にエディタ、右側にAIチャットの2ペイン構造。
* **バックエンド**: APIサーバー (Spring Boot / FastAPI等) - ユーザー管理、学習データ管理、Docker制御。
* **実行環境**: サンドボックス (Dockerコンテナ) - ユーザーのコードを隔離環境で実行し、標準出力とエラーを取得。
* **AI API**: OpenAI API等 - 問題の自動生成、およびソクラテスメソッド（問答法）によるヒント提示。

---

## 3. 生徒側機能要件 (フロントエンド)

### 3.1 コード実行と正誤判定フロー
1. AIが生成した「問題文・初期コード・期待される標準出力」をユーザーに提示。
2. ユーザーがコードを記述し「実行」を押下。
3. バックエンドのDockerコンテナ内でコードをコンパイル・実行。
4. **システム判定**: 実行結果（標準出力）と期待される出力をDiff（差分比較）し、正誤を判定する（判定にAIは使用しない）。

### 3.2 AIチューター機能 (サイドバー)
* **発火条件**: ユーザーが任意で「AIに助けを求める」ボタンを押下した際、またはサイドバーからチャットを送信した際。
* **プロンプト制限**: AIは**絶対に正解コードを直接提示しない**。コンパイルエラーの読み方や、論理エラーのデバッグ方法など、ユーザーに気づきを与えるヒントのみを出力する。

### 3.3 学習進行: スキルツリー（タグ熟練度）システム
固定のコースやカリキュラムを廃止し、タグ（概念）ごとの経験値（EXP）によるアンロック方式を採用する。
* **UI**: ノードとエッジで構成されたスキルツリー画面。
* **アンロック条件**: 例「変数」がLv.3に達すると、「条件分岐」が解放される。
* **問題の動的生成**: ユーザーの現在の解放済みタグの組み合わせから、適切な難易度の問題をAIが動的に生成する。

### 3.4 EXP（経験値）獲得ロジック
AIの生成コード判定が困難なため、客観的指標によるスコアリングを実施する（最大30pt）。
* **基本クリア**: +10 pt (テストケース全通過)
* **一発正解**: +5 pt (1回目の提出でクリア)
* **ノーヒント**: +5 pt (AIへのヒントリクエスト回数が0回)
* **クリーンコード**: +5 pt (静的解析ツール/LinterでのWarningが0件)
* **実行効率**: +5 pt (実行時間・メモリ使用量が基準値以下)

## 4. 運営側機能要件 (管理画面ダッシュボード)

AIが動的に生成したコンテンツの品質を担保し、資産化するための機能群。

### 4.1 コンテンツ（問題）承認フロー
* **ステータス管理**: 問題データを `AUTO_GENERATED`(未確認)、`APPROVED`(承認済)、`ARCHIVED`(破棄) で管理。
* **検証機能**: 管理画面上でAI生成問題の「テスト実行」を行い、模範解答がテストケースをクリアするかシステム検証する機能。
* **承認**: 検証済みの良問を承認し、全ユーザーの公開問題プールへ追加する。

### 4.2 スキルツリー保守
* **ツリーエディタ**: タグの追加、およびタグ間の依存関係（解放条件）の紐付け設定。
* **EXP調整**: 各レベルに必要な経験値テーブルの設定。

### 4.3 分析・システム監視
* ユーザーの停滞ポイント（離脱率の高い問題、エラーが多い箇所）の特定。
* AI APIのトークン消費量・コスト監視、Dockerコンテナのリソース稼働状況の監視。

## 5. データモデル概要 (主要テーブル)

| テーブル名 | 用途・主要カラム |
| :--- | :--- |
| `users` | ユーザー基本情報 |
| `tags` | スキルタグ情報 (ID, name, category) |
| `user_tag_progress` | ユーザー毎のスキル進捗 (user_id, tag_id, level, exp) |
| `tag_unlock_conditions`| ツリーの依存関係 (target_tag_id, required_tag_id, required_level) |
| `problems` | 問題データ (title, description, test_cases, solution, **status**) |
| `submissions` | 提出履歴 (user_id, problem_id, code, result, exp_earned) |
