# 01. プロジェクト憲章とロードマップ

## 1. ビジョン
「実務で自走できるエンジニア」を育成するため、AI駆動型のOJTシミュレーションを提供。従来の「動けば正解」を卒業し、コード品質、リーディング、テスト設計までをカバーする。

---

## 2. ロードマップ

### Phase 1: MVP（コア学習体験）
- **多言語対応スキルツリー**: 言語ごとに独立したスキルツリーを管理。管理者が言語を追加・有効化できる。
- **コード実行サンドボックス**: Docker + gVisor による安全な実行環境。タイムアウト2秒・メモリ128MB・ネットワーク完全遮断。
- **二段階の判定方式**: 問題ごとに `judgment_type` を設定し切り替える。
  - `STDOUT`：標準出力と期待値の Diff 比較（基礎問題向け）
  - `TESTCASE`：関数呼び出し（assert）によるテストケース検証（応用問題向け）
- **エラーの日本語化**: コンパイルエラー・実行時エラー・タイムアウトを種別区別し、日本語メッセージ＋行番号ハイライト情報付きで返却。
- **非同期実行フロー**: POST /execute → task_id 即時返却 → 500ms間隔のポーリングで結果取得（最大10秒）。
- **AIヒント**: ソクラテスメソッドによる段階的ヒント。正解コードは絶対に直接提示しない。
- **プレースメントテスト**: 経験者向けの一括スキルアンロック。
- **管理者による問題承認フロー**: AUTO_GENERATED → テスト実行 → APPROVED の3ステータス管理。
- **スキルテンプレート基盤（Phase 2準備）**: `tags.template_id` をNULL許容で定義。Phase 2でのAI一括展開に備える。

### Phase 2: 実務・品質
- **テストフレームワーク対応**: JUnit（Java）・pytest（Python）・Jest（JavaScript）などのユニットテストフレームワークを使った課題形式。
- **バグハント（リーディング）課題**: 意図的にバグを埋め込んだコードを読んで修正する課題。マイルストーンクリア後に解放。
- **擬似PRレビュー＆リテイク**: テスト通過後にAIシニアエンジニアがコードレビューを実施。指摘事項を修正・再チェックしてはじめて完全クリア。
- **スキルテンプレートとAI一括展開**: AIが共通スキル概念を検出し、管理者の承認で全言語に一括展開。
- **弱点プロファイルの蓄積**: ユーザーの頻出エラー傾向をプロファイル化し、AIプロンプトに動的注入。

### Phase 3: 設計・フルスタック
- **テストコード作成課題**: ユーザー自身がJUnit等のテストコードを書く課題。学習割合を「実装50%：テスト/リーディング50%」へ移行。
- **上流工程シミュレーション**: 基本設計・詳細設計書からのシステム構築体験。
- **ローカル環境移行支援**: VSCode / Eclipse等への環境エクスポート機能。

### Phase 4以降: 発展的テスト手法（将来構想）
- **Excelベースのデータ駆動テスト**: Excelファイルでテストケースを定義し、大量入出力パターンを課題として実行・検証できる判定エンジン拡張。
- **UIテスト課題**: Selenium / Playwright等を使ったブラウザUIテストを課題として実行できる環境。フロントエンドエンジニア育成に特化。

## 3. ゲームデザイン（熟練度システム）

### EXP獲得ロジック（最大30pt/問）
| 項目 | 加点 | 条件 |
|---|---|---|
| 基本クリア | +10pt | テストケース全通過。同一タグ・難易度の連続解答で減衰（100%→50%→20%→固定1pt） |
| 一発正解 | +5pt | 1回目の提出でクリア |
| ノーヒント | +5pt | AIヒント要求回数が0回 |
| クリーンコード | +5pt | Linter警告ゼロ |
| 実行効率 | +5pt | 実行時間・メモリ使用量が基準値以下 |

### スキルツリー解放
- 前提スキルの指定レベル到達により次スキルが解放される。
- 言語ごとに独立したツリーを持つ。
- 目標から逆算した学習ルートの表示（Phase 2以降）。
